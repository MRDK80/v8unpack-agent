# Участие в разработке

Спасибо за интерес к проекту! Этот документ объясняет, как предложить
исправление или новую функциональность.

## Требования

- Python 3.10 или новее
- Git

## Быстрый старт

```bash
# 1. Форкните репозиторий на GitHub, затем клонируйте свой форк
git clone https://github.com/<ваш-логин>/v8unpack-agent.git
cd v8unpack-agent

# 2. Установите пакет в режиме разработки вместе с зависимостями для тестов
pip install -e ".[test]"

# 3. Убедитесь, что тесты проходят
pytest
```

## Рабочий процесс

1. Создайте ветку от `main` с говорящим именем:
   ```bash
   git checkout -b fix/form-artifact-warnings   # для исправлений
   git checkout -b feat/erf-batch-unpack         # для новых функций
   git checkout -b docs/update-readme            # для документации
   ```
2. Внесите изменения. Если добавляете новое поведение — добавьте тест.
3. Убедитесь, что тесты зелёные: `pytest`
4. Запушьте ветку и откройте Pull Request в `main`.

## Соглашение об именах веток

| Префикс | Когда использовать |
|---|---|
| `fix/` | Исправление бага |
| `feat/` | Новая функциональность |
| `docs/` | Только документация |
| `refactor/` | Рефакторинг без изменения поведения |
| `test/` | Только тесты |

## Соглашение о коммитах

Используйте префиксы в стиле [Conventional Commits](https://www.conventionalcommits.org/):

```
fix: корректно обрабатывать Form.bin с пустым Items/
feat: добавить пакетную распаковку .erf
docs: обновить примеры в README
test: покрыть is_form_stale граничными случаями
refactor: вынести логику mtime в отдельный модуль
```

## Требования к Pull Request

- Все 4 CI-джобы должны быть зелёными (ubuntu + windows, py3.10 + py3.12).
- Новая публичная функция — значит новый тест. Тесты синтетические: реальный
  контейнер 1С не нужен, используйте временные файловые деревья и
  заглушку-распаковщик (см. существующие тесты в `tests/`).
- Описание PR объясняет *что* изменилось и *почему*.

## Запуск тестов

```bash
# Все тесты
pytest

# С подробным выводом
pytest -v

# Конкретный файл
pytest tests/test_form_artifact.py
```
